# Tashkent PM2.5 — Air-Quality Attribution & Forecasting

A reproducible, machine-learning study of fine-dust pollution (**PM2.5**) in **Tashkent,
Uzbekistan**. It answers two questions: **where does the bad air come from — nature or
ourselves?** and **can we warn people, ahead of time, when it will be dangerous?**

**Author:** Imronbek Shoniyozov

This repository holds the **data, models, and analysis code** behind the study (hypothesis tests,
ground-truth validation, source attribution, and the forecasting tool). The write-up itself is
published separately.

## What's inside

- **`src/`** — the analysis and modelling pipeline (one script per step).
- **`data/`** — cached inputs and processed daily dataset.
- **`models/`** — trained models and saved metrics.
- **`figures/`** — result figures, regenerated from the scripts.
- **`config.py`** — study coordinates, thresholds, and split dates.

## Headline findings

- **The pollution is mostly man-made.** A source-fingerprint analysis attributes roughly
  **55%** of the average PM2.5 to **mazut and coal heating**, at most **~16%** to traffic, and
  **~30%** to a natural background — so about **70% is locally produced**. **Dust storms are not
  the cause** (the dustiest days are among the cleanest).
- **A popular global model (CAMS) reads about half the real value** and misses ~80% of dangerous
  days — a bias confirmed at nine sensors across five cities; a calibrated model corrects it.
- **A warning tool** outputs calibrated probabilities and a 7-day risk outlook — reliable for the
  next few days, fading into a probabilistic weekly outlook beyond that.

## Reproduce it

```bash
pip install -r requirements.txt          # pandas, numpy, scikit-learn, lightgbm, statsmodels, shap, matplotlib, requests
cp .env.example .env                      # then add your own free API tokens (see below)
python src/collect.py                     # fetch the public data (Open-Meteo / OpenAQ)
python src/features.py                    # build the daily feature table
python src/train_ground_truth.py          # the calibrated forecaster
python src/forecast_week.py               # live 7-day outlook for Tashkent
python src/verify_claims.py               # recompute every headline number from source (28/28)
```

Individual analyses (each is standalone): `research.py`, `mazut_hypothesis.py`,
`mazut_causal.py`, `source_breakdown.py`, `validate_regional_sensors.py`,
`scenario_evaluation.py`, `horizon_levels.py`, `adaptive_confidence.py`,
`probabilistic_forecast.py`, `tropomi_so2.py` (satellite SO₂ cross-check),
`verify_claims.py` (recompute every headline number — 28/28).

## Live danger alerts (Telegram)

A warning bot — [**@airqualitytash_bot**](https://t.me/airqualitytash_bot) — notifies
subscribers when a dangerous PM2.5 level is forecast within the week. Users press **Start**
to subscribe (`/stop` to leave); the bot can only message people who have started it.

```bash
# add TELEGRAM_BOT_TOKEN=... to .env (from @BotFather), then run daily:
python src/telegram_alert.py            # sync subscribers, forecast, alert if a dangerous day is coming
python src/telegram_alert.py --status   # show the outlook + danger tier, send nothing
python src/telegram_alert.py --force     # broadcast the current outlook to all (manual)
```

It de-duplicates (one alert at onset/escalation + one all-clear, never daily spam). Run it
once a day with cron, e.g. `0 8 * * * cd /path/to/TashkentAir && python src/telegram_alert.py`.

## API keys

All data comes from **free** public APIs. Keys live in a local **`.env`** file (which is
git-ignored and never committed). Copy `.env.example` and fill in your own:

```
WAQI_TOKEN=...        # aqicn.org/data-platform/token (free)
OPENAQ_TOKEN=...      # openaq.org (free)
IQAIR_TOKEN=...       # iqair.com Community API (free, optional)
```

Open-Meteo (weather, CAMS air-quality, forecasts) needs no key.
